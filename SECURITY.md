## 报告漏洞

如果您发现安全漏洞，请通过以下方式报告：

1. **不要**通过公开的 GitHub Issues 报告安全问题
2. 发送邮件至 [your-email@example.com]
3. 提供漏洞的详细描述
4. 包含复现步骤（如果可能）

## 响应时间

- 我们会在 48 小时内确认收到您的报告
- 我们会在 7 天内提供初步评估
- 我们会在 30 天内发布修复版本

## 安全最佳实践

### 模型安全

1. **输入验证**: 始终验证用户输入
2. **输出过滤**: 对模型输出进行适当过滤
3. **访问控制**: 限制模型访问权限
4. **日志记录**: 记录所有模型调用

### 数据安全

1. **数据加密**: 加密存储敏感数据
2. **数据匿名化**: 匿名化处理训练数据
3. **数据备份**: 定期备份重要数据
4. **数据删除**: 及时删除不再需要的数据

### 部署安全

1. **环境隔离**: 使用容器隔离部署环境
2. **网络限制**: 限制网络访问权限
3. **密钥管理**: 安全管理 API 密钥
4. **监控告警**: 设置安全监控和告警
